## Section 2: employee.proto

```protobuf
syntax = "proto3";

package company;

option java_package = "com.example.company";

// Departments in the organization
enum Department {
  DEPARTMENT_UNKNOWN = 0;   // proto3 requires 0 as default
  ENGINEERING = 1;
  MARKETING = 2;
  SALES = 3;
  HR = 4;
  FINANCE = 5;
}

// A single employee record
message Employee {
  string employee_id    = 1;   // unique identifier e.g. "EMP001"
  string name           = 2;
  int32  age            = 3;
  Department department = 4;
  double salary         = 5;
  bool   is_active      = 6;
  string email          = 7;
  repeated string skills = 8;  // list of skill tags
}

// Wrapper for multiple employees (used to write a list to one file)
message EmployeeList {
  repeated Employee employees  = 1;
  int32            total_count = 2;
}
```

**Why a wrapper message?** Protobuf does not have a top-level array type. Wrapping with `EmployeeList` lets you serialize the whole collection as one contiguous blob.

## Section 3: Generating Python Code

Run protoc from the directory containing `employee.proto`:

```bash
protoc --python_out=. employee.proto
```

This creates `employee_pb2.py`. The `_pb2` suffix is a protobuf convention (pb = protocol buffer, 2 = second-generation API).

**What's inside `employee_pb2.py`:**
- `DESCRIPTOR` — the file's schema metadata
- `Department` enum class with integer constants (`ENGINEERING = 1`, etc.)
- `Employee` class with getters/setters for every field
- `EmployeeList` class

**Never edit `employee_pb2.py` manually.** If you change `employee.proto`, regenerate it.

### Importing in your script

```python
from employee_pb2 import Employee, EmployeeList, Department
```

## Section 4: Creating and Populating a Message

```python
from employee_pb2 import Employee, Department

# Method 1: create empty, then assign fields
emp = Employee()
emp.employee_id = "EMP001"
emp.name        = "Alice Johnson"
emp.age         = 28
emp.department  = Department.ENGINEERING
emp.salary      = 95000.0
emp.is_active   = True
emp.email       = "alice@company.com"
emp.skills.extend(["Python", "Go", "Kubernetes"])

# Method 2: keyword arguments at construction time (cleaner)
emp = Employee(
    employee_id = "EMP001",
    name        = "Alice Johnson",
    age         = 28,
    department  = Department.ENGINEERING,
    salary      = 95000.0,
    is_active   = True,
    email       = "alice@company.com",
)
emp.skills.extend(["Python", "Go", "Kubernetes"])
```

**Note on repeated fields:** You cannot assign a list directly (`emp.skills = [...]` raises an error). Use `.extend()` for multiple items or `.append()` for one.

```python
emp.skills.append("Docker")           # add one
emp.skills.extend(["AWS", "Linux"])   # add many
```

## Section 5: Serialization

### SerializeToString()

```python
data = emp.SerializeToString()

print(type(data))   # <class 'bytes'>
print(len(data))    # ~62 bytes for Alice's record
```

### Comparing size to JSON

```python
import json
from google.protobuf.json_format import MessageToJson

json_str = MessageToJson(emp)
print(f"Protobuf size: {len(data)} bytes")
print(f"JSON size:     {len(json_str.encode())} bytes")
# Protobuf size: 62 bytes
# JSON size:     183 bytes
```

Protobuf is roughly **3× smaller** for this record. The gap widens with more numeric data.

### Hex dump of the binary

Here is what Alice's serialized bytes look like (first ~30 bytes explained):

```
0a 07 45 4d 50 30 30 31    field 1 (employee_id), len=7, "EMP001"
12 0d 41 6c 69 63 65 20    field 2 (name), len=13, "Alice..."
4a 6f 68 6e 73 6f 6e       "...Johnson"
18 1c                      field 3 (age), varint=28
20 01                      field 4 (department), varint=1 (ENGINEERING)
```

**Wire type encoding:**
- Tag byte = `(field_number << 3) | wire_type`
- `0a` = field 1 (`1 << 3 = 8 = 0x08`), wire type 2 (`0x08 | 0x02 = 0x0a`)
- Wire type 0 = varint, 1 = 64-bit, 2 = length-delimited, 5 = 32-bit

## Section 6: Writing to File

```python
from employee_pb2 import Employee, EmployeeList, Department

# Build a list of employees
emp_list = EmployeeList()

employees_data = [
    ("EMP001", "Alice Johnson",  28, Department.ENGINEERING, 95000.0, True,  "alice@co.com"),
    ("EMP002", "Bob Smith",      35, Department.MARKETING,   72000.0, True,  "bob@co.com"),
    ("EMP003", "Charlie Davis",  42, Department.HR,           68000.0, False, "charlie@co.com"),
]

for emp_id, name, age, dept, salary, active, email in employees_data:
    emp = emp_list.employees.add()
    emp.employee_id = emp_id
    emp.name        = name
    emp.age         = age
    emp.department  = dept
    emp.salary      = salary
    emp.is_active   = active
    emp.email       = email

emp_list.total_count = len(emp_list.employees)

# Write to binary file — MUST use "wb" (binary write mode, not "w")
with open("employees.bin", "wb") as f:
    f.write(emp_list.SerializeToString())

print(f"Saved {emp_list.total_count} employees.")
```

> **Common mistake:** Using `open("employees.bin", "w")` (text mode) corrupts binary data on Windows because it translates `\n` bytes. Always use `"wb"`.

## Section 7: Reading from File

```python
from employee_pb2 import EmployeeList

# Read raw bytes from file — MUST use "rb" (binary read mode)
with open("employees.bin", "rb") as f:
    raw_bytes = f.read()

# Deserialize
emp_list = EmployeeList()
emp_list.ParseFromString(raw_bytes)

# Access the data
print(f"Total employees: {emp_list.total_count}")

for emp in emp_list.employees:
    print(f"\n{emp.name} ({emp.employee_id})")
    print(f"  Age:        {emp.age}")
    print(f"  Department: {Department.Name(emp.department)}")
    print(f"  Salary:     ${emp.salary:,.2f}")
    print(f"  Active:     {emp.is_active}")
    print(f"  Skills:     {', '.join(emp.skills)}")
```

`Department.Name(emp.department)` converts the integer back to the enum name string (e.g. `1` → `"ENGINEERING"`).

## Section 8: JSON Serialization

Protobuf has a canonical JSON mapping. This is useful when you need to inspect data or bridge to a REST API.

```python
from google.protobuf.json_format import MessageToJson, Parse
from employee_pb2 import Employee, Department

emp = Employee(
    employee_id="EMP001", name="Alice Johnson",
    department=Department.ENGINEERING, salary=95000.0
)

# Protobuf → JSON string
json_str = MessageToJson(emp)
print(json_str)
```

Output:
```json
{
  "employeeId": "EMP001",
  "name": "Alice Johnson",
  "department": "ENGINEERING",
  "salary": 95000.0,
  "isActive": false
}
```

Note: proto3 uses **camelCase** in JSON by default (field `employee_id` → key `"employeeId"`).

```python
# JSON string → protobuf object
emp2 = Parse(json_str, Employee())
print(emp2.name)   # Alice Johnson
```

## Section 9: Inspecting the Binary Format

### Wire types

Every field in a protobuf message is encoded with a **tag** followed by its value. The tag encodes both the field number and the wire type.

| Wire Type | ID | Used For |
|---|---|---|
| Varint | 0 | int32, int64, uint32, uint64, sint32, sint64, bool, enum |
| 64-bit | 1 | fixed64, sfixed64, double |
| Length-delimited | 2 | string, bytes, embedded messages, repeated fields |
| 32-bit | 5 | fixed32, sfixed32, float |

### Tag calculation

```
tag_byte = (field_number << 3) | wire_type
```

For `employee_id` (field 1, string = wire type 2):
```
tag = (1 << 3) | 2 = 10 = 0x0a
```

For `age` (field 3, int32 = wire type 0):
```
tag = (3 << 3) | 0 = 24 = 0x18
```

### Walking through bytes

```python
emp = Employee(employee_id="EMP001", name="Alice", age=28)
data = emp.SerializeToString()
print(data.hex())
# 0a 06 454d5030303112 05416c69636518 1c
#
# 0a        = tag for field 1 (employee_id), wire type 2
# 06        = length: 6 bytes
# 454d503030 31 = "EMP001" in ASCII
# 12        = tag for field 2 (name), wire type 2
# 05        = length: 5 bytes
# 416c696365 = "Alice" in ASCII
# 18        = tag for field 3 (age), wire type 0
# 1c        = varint 28
```

## Key Takeaways

- Generate Python code with `protoc --python_out=. your.proto` — never edit `_pb2.py`
- `SerializeToString()` → bytes; `ParseFromString(bytes)` → object
- Always open files in binary mode: `"wb"` to write, `"rb"` to read
- Use a wrapper message (like `EmployeeList`) to store multiple records in one file
- `MessageToJson()` and `Parse()` bridge between protobuf and JSON
- Protobuf binary is typically 3–10× smaller than equivalent JSON

## Common Errors

| Error | Cause | Fix |
|---|---|---|
| `ModuleNotFoundError: employee_pb2` | Haven't run protoc yet | Run `protoc --python_out=. employee.proto` |
| `UnicodeDecodeError` when reading file | Opened file in text mode | Change `open(..., "r")` to `open(..., "rb")` |
| `emp.skills = [...]` raises `AttributeError` | repeated fields are not plain lists | Use `emp.skills.extend([...])` |
| Fields missing in JSON output | proto3 omits default/zero values | Use `MessageToJson(emp, including_default_value_fields=True)` |
| `ParseFromString` returns 0 / no data | Passed empty bytes or text string | Ensure you read the file in binary mode and file is non-empty |
